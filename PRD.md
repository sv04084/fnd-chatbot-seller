# PRD

## 개요
`fnd-chatbot-seller`는 현재 구현 코드가 없는 초기 상태의 프로젝트입니다. 저장소에는 `README.md`만 존재하며, 실행 가능한 애플리케이션 코드, 설정 파일, 패키지 매니저 파일, API 정의, 데이터 모델이 아직 포함되어 있지 않습니다.

이 문서는 현재 저장소 상태를 기준으로, 향후 이 프로젝트가 판매자용 챗봇 서비스로 확장될 가능성을 고려한 최소 요구사항 문서 초안입니다.

## 현재 상태
- 저장소 내 확인된 파일: `README.md`
- 미확인 항목:
  - 프론트엔드 코드
  - 백엔드 코드
  - 데이터베이스 스키마
  - 배포 설정
  - 테스트 코드
  - 환경 변수 예시 파일

## 제품 목표
- 판매자 업무를 보조하는 챗봇 서비스의 기초 구조를 마련한다.
- 향후 대화형 문의 응답, 상품/주문/고객 응대, 운영 FAQ 자동화에 확장 가능하도록 설계한다.
- 초기 단계에서는 빠른 프로토타이핑과 요구사항 정리를 우선한다.

## 대상 사용자
- 판매자 운영 담당자
- CS/상담 담당자
- 내부 운영 관리자

## 핵심 문제
- 현재 저장소에는 제품 동작을 설명할 실제 구현이 없다.
- 기능 범위, 기술 스택, 데이터 흐름, 운영 정책이 정의되지 않았다.
- 이후 구현이 시작될 경우 문서와 코드 간 불일치가 빠르게 발생할 수 있다.

## MVP 범위 제안
- 판매자 질문 입력 UI
- 챗봇 응답 UI
- 기본 대화 세션 관리
- FAQ 또는 사전 정의 응답 연결
- 관리자/운영자용 설정 포인트 정의
- 로그 확인이 가능한 최소한의 백엔드 또는 서버리스 구조

## 비기능 요구사항
- 문서 우선: 구현 전 요구사항과 제약을 먼저 명시
- 추적 가능성: 주요 결정은 문서에 기록
- 보안: API 키, 고객 데이터, 판매자 계정정보는 저장소에 커밋 금지
- 확장성: 추후 외부 LLM, 벡터 검색, CRM/주문 시스템 연동 가능 구조 고려

## 성공 기준
- 프로젝트 실행 방법이 문서화되어 있다.
- 최소 1개의 사용자 플로우가 구현되어 있다.
- 환경 변수와 로컬 실행 절차가 정리되어 있다.
- 주요 제약사항이 `Agents.md`에 유지된다.

## 향후 필요한 산출물
- 기술 스택 결정 문서
- 화면/기능 명세
- API 명세
- 데이터 모델 초안
- 운영 제약 및 보안 정책
- 테스트 전략

## 현재 기준 결정 사항
- 본 프로젝트는 아직 구현 전 단계로 판단한다.
- 이후 코드가 추가되면 본 문서는 실제 구현 상태에 맞게 갱신되어야 한다.
