# Agents

## 목적
이 문서는 이 저장소에서 작업하는 에이전트와 개발자가 반드시 확인해야 하는 운영 규칙, 제약사항, 문서 갱신 규칙을 정의합니다.

## 현재 저장소 상태
- 현재 확인된 파일은 `README.md`, `PRD.md`, `Agents.md`뿐입니다.
- 구현 코드가 없으므로, 추정에 기반한 단정은 금지합니다.
- 새로운 코드가 추가되면 이 문서의 제약사항과 작업 규칙을 함께 갱신해야 합니다.

## 작업 원칙
- 실제 코드와 설정 파일을 우선 근거로 삼습니다.
- 코드에 없는 요구사항은 반드시 가정으로 표시합니다.
- 민감정보는 저장소에 커밋하지 않습니다.
- 실행 방법, 환경 변수, 외부 연동 정보는 문서화합니다.

## 현재 제약사항
- 애플리케이션 구조 미정
- 프론트엔드/백엔드 스택 미정
- 데이터 저장소 미정
- 테스트 전략 미정
- 배포 방식 미정
- 인증/권한 모델 미정

## 문서 자동 업데이트 규칙
아래 상황이 발생하면 `Agents.md`를 즉시 갱신합니다.
- 새로운 필수 환경 변수가 추가된 경우
- 외부 API 또는 데이터베이스가 추가된 경우
- 배포 방식 또는 실행 방식이 변경된 경우
- 보안상 주의가 필요한 정보가 생긴 경우
- 특정 라이브러리/플랫폼 제약으로 구현 방식이 제한된 경우
- 사용자 요청으로 반드시 지켜야 하는 프로젝트 규칙이 생긴 경우

## 에이전트용 업데이트 프롬프트
아래 프롬프트를 기준으로 작업 중 제약사항을 발견하면 `Agents.md`를 자동 갱신합니다.

```text
작업 중 새 제약사항, 운영 규칙, 보안 주의점, 환경 변수 요구사항, 외부 서비스 의존성, 배포/실행 방식 변경이 확인되면 즉시 Agents.md를 업데이트하라.

업데이트 원칙:
1. 추정이 아닌 실제 코드/설정/사용자 명시 요구만 반영한다.
2. 기존 항목과 충돌하면 최신 사실 기준으로 수정한다.
3. 민감정보 자체는 기록하지 않고, 민감정보가 필요하다는 사실과 관리 원칙만 기록한다.
4. 사용자가 반드시 따라야 하는 실행 조건이 생기면 "현재 제약사항" 또는 별도 섹션에 명시한다.
5. 기능 추가 후 문서가 코드보다 뒤처지지 않도록 같은 작업 흐름 안에서 함께 수정한다.
```

## 문서 작성 규칙
- 짧고 명확하게 작성합니다.
- 구현 사실과 계획을 구분합니다.
- 파일명, 환경 변수명, 서비스명은 정확히 표기합니다.
- 코드 구조가 생기면 디렉토리 기준으로 책임 범위를 정리합니다.

- 데이터베이스 종류 및 접속 방식
- 테스트/배포 절차
